Aside: A note on terminology.

The text below refers to the foundational system that it is presenting as ‚Äúhomotopy type theory‚Äù, while it would be technically more correct to refer to it as a homotopy type theory. HoTT is not a specific type theory, but a family of type theories that put emphasis on the homotopy-theoretic interpretation of type theory. In particular, a homotopy type theory is expected to validate univalence and have some class of higher inductive types.

Below, we describe cubical type theory, which is one particular homotopy type theory. In cubical type theory, we interpret types as the Kan complices in the category of De Morgan cubical sets, PSh(‚ñ°)\mathrm{PSh}(\square)PSh(‚ñ°) . Type theoryüîó

Warning: The text below is subject to change! It‚Äôs still very much a document in flux. In particular, the type theory section doesn‚Äôt talk about induction, which is something that will be expanded on in the future.

Type theory is a foundational system for mathematics which, in contrast to a set-theoretic foundation like ZFC, formalises mathematical constructions rather than mathematical proofs. That is, instead of specifying which logical deductions are valid, and then giving a set of axioms which characterise the behaviour of mathematical objects, a type-theoretic foundation specifies directly which mathematical constructions are valid.

Formally speaking, it is impossible to construct objects in set theoretic foundations; Rather, one can, by applying the deduction rules of the logical system and appealing to the axioms of the set theory, prove that an object with the desired properties exists. As an example, if we have a way of referring to two objects xxx and yyy , we can prove that there exists a set representing the ordered pair (x,y)(x, y)(x,y) , using the axiom of pairing. Writing quasi-formally, we could even say that this set is constructed using the expression {x,{x,y}}\{x, \{x, y\} \}{x,{x,y}} , but this is only a notational convenience.

In contrast, a type-theoretic foundational system, such as HoTT (what we use here), specifies directly how to construct mathematical objects, and proving a theorem becomes a special case of constructing an object. To specify what constructions are well-behaved, we sort the mathematical objects into boxes called types. Just like in set theory, the word ‚Äúset‚Äù is a primitive notion, in type theory, the word ‚Äútype‚Äù is a primitive notion. However, we can give an intuitive explanation: ‚ÄúA type is a thing that things can be‚Äù. For instance, things can be natural numbers, but things can not be ‚Äútwo‚Äù ‚Äî hence, the natural numbers are a type, but two is not. Formally, a type is specified by giving:

  • A formation rule, which tells us which expressions denote valid types at all. For instance, the formation rule for product types tells us that N√óN\mathbb{N} \times \mathbb{N}N√óN denotes a well-formed type, but ‚ÄúN√ó‚Üí\mathbb{N} \times \toN√ó‚Üí ‚Äù does not. The formation rules also have the duty of preventing us from running into size issues like Russell‚Äôs paradox, by constraining the use of universes.

  • A handful of introduction rules, which tell us how to piece previously-constructed objects together to form an object of a different type. Sticking with products as an example, if we have previously constructed objects called aaa and bbb , we can construct an ordered pair (a,b)(a, b)(a,b) .

    Since the introduction rules are inseparable from their types, it is impossible to construct an object without knowing in which type it lives. If we used the rule which introduces ordered pairs, we know for sure that we built an object of the product type. To specify that an object lives in a certain type, we use the notation a:Aa : Aa:A ‚Äî which can be read as ‚Äúaaa inhabits AAA ‚Äù, or ‚Äúaaa is a point of AAA ‚Äù, or ‚Äúaaa has type AAA ‚Äù.

    The notation for stating inhabitation agrees with mathematical parlance in when both make sense. For example, a function from the reals to the reals is commonly introduced with the notation f:R‚ÜíRf : \mathbb{R} \to \mathbb{R}f:R‚ÜíR . In type theory, this is made formal as a typing declaration: fff is a point of the type of functions R‚ÜíR\mathbb{R} \to \mathbb{R}R‚ÜíR , hence we know that the rules of function types can be used with fff . However, when introducing a more complex object, we‚Äôre forced to break into informality and conventional shorthand: ‚ÄúLet G be a group‚Äù. In HoTT, we can directly write G:GroupG : \mathrm{Group}G:Group 1, which tells us that the rules for the type of groups apply to GGG .

  • A handful of elimination rules, which say how inhabitants of that type can be manipulated. Keeping with the running example, the product type has elimination rules corresponding to projecting either the first or second component of a pair.

    Furthermore, each elimination rule comes with specified behaviour for when it ‚Äúinteracts‚Äù with the introduction rules, a process which we generally call reduction. Repeatedly applying these ‚Äúinteractions‚Äù tells us how a given construction can have computational behaviour ‚Äî by performing a delicate dance of elimination and introduction, we can go from 2+22 + 22+2 to 444 .

Now that we have an intuitive motivation for the notions of type theory, we can more precisely discuss the structure of formal type theory and how it differs from formal (material) set theory. The first (and perhaps most important) difference is that formal set theory is a ‚Äútwo layered‚Äù system, but type theory only has ‚Äúone layer‚Äù. In (for example) ZFC, we have the deductive framework of first-order logic (the ‚Äúfirst layer‚Äù), and, using the language of that logical system, we assert the axioms of set theory. This remains valid if we switch set theories: For instance, IZF would substitute first-order logic with first-order intuitionistic logic, and remove the assertion that the axiom of choice holds.

In contrast, type theory only has one layer: The layer of types. This segues nicely into another difference between type theory and set theory, which concerns the setup of their deductive systems. A deductive system is a specification for how to derive judgements using rules. We can think of the deductive system as a game of sorts, where the judgements are to be thought of as board positions, and the rules specify the valid moves.

In the deductive system of first-order logic, there is only one kind of judgement, stating that a proposition has a proof. If AAA is a proposition, then the judgement ‚ÄúAAA has a proof‚Äù is written ‚Äú‚ä¢A\vdash A‚ä¢A ‚Äù. Note that the judgement ‚ä¢A\vdash A‚ä¢A ‚Äã is a part of the metatheory, and the proposition AAA is a part of the theory itself. If AAA is a type that denotes a proposition, then this is analogous to the judgement ‚ä¢A\vdash A‚ä¢A of first-order logic, and we refer to aaa as a witness of AAA , or simply as a proof of AAA . Even better, if AAA is a purely logical proposition such that ‚ä¢A\vdash A‚ä¢A holds without any appeal to the principle of excluded middle, then there is a term aaa for which ‚ä¢a:A\vdash a : A‚ä¢a:A holds in type theory.

When AAA is being thought of as a collection of things, we might imagine that a:Aa : Aa:A is analogous to a‚ààAa \in Aa‚ààA . The third and final difference is the treatment of equality, which is arguably also the most important part of homotopy type theory. In set theory, given sets aaa and bbb , we have a proposition a=ba = ba=b , which is given meaning by the axiom of extensionality: a=ba = ba=b whenever aaa and bbb behave the same with respect to ‚àà\in‚àà . Since definitional equality is a judgement, it‚Äôs also not subject to the internal propositional connectives ‚Äî we can not prove, disprove, assume or negate it when working inside type theory, for it does not make sense to say ‚Äúif aaa and bbb are equal by definition, then [‚Ä¶]‚Äù. Correspondingly, in the Agda formalisation, definitional equality is invisible, since it‚Äôs represented by the computation rules of the type theory.

Interlude: Basics of Pathsüîó

Since the treatment of paths is the most important aspect of homotopy type theory, rest assured that we‚Äôll talk about it in more detail later. However, before discussing the dependent sum of a type family, we must discuss the fundamentals of paths, so that the categorical/homotopical connections can be made clear. Before we even get started, though, there is something that needs to be made very clear:

Paths are not equality!

The only conceptualisation of paths that is accurate in all cases is that types are ‚àû\infty‚àû -groupoids, and hence path spaces are the Hom-‚àû\infty‚àû -groupoids given by the structure of each type. However, this understanding is very bulky, and doesn‚Äôt actually help understanding paths all that much ‚Äî so we present a bunch of other analogies, with the caveat that they only hold for some types, or are only one of the many possible interpretations of paths in those types.

The ideas presented below are grouped in blue <details> elements, but not because they aren‚Äôt important; You should read all of them.

Idea #1: Some types behave like sets, providing an embedding of ‚Äúclassical‚Äù equality behaviour into homotopy type theory. While these types may not be as ‚Äúexciting‚Äù as others, they aren‚Äôt any less important!

Contradicting the big bold warning up there, in a very restricted class of types, paths do behave like the equality relation! We call these types sets, and their defining property is that they only have identity paths ‚Äî hence they could also be called ‚Äúdiscrete types‚Äù, by analogy with ‚Äúdiscrete category‚Äù, but this terminology is not employed because ‚Äúdiscrete‚Äù already has a distinct type-theoretic meaning.7

Types in this class include the natural numbers, the integers, the rationals and the reals; Also included is every finite type, and the powerset of any type at all. The type of functions A‚ÜíBA \to BA‚ÜíB , where BBB is a set, is again a set. We can generalise this and say that ‚Äúmost‚Äù type formers preserve set-ness.

Note that the meaning of ‚Äúis a set‚Äù/‚Äúis not a set‚Äù in HoTT is distinct from how that sentence is used in classical mathematics, where a collection is a set if it‚Äôs definable in the language of whatever foundational system you‚Äôre using. For example, in ZFC, the collection of all sets is not a set ‚Äî because if it were, we could reproduce Russell‚Äôs paradox.

In HoTT, being a set has to do with spatial structure: a set has only trivial paths. Examples of non-sets would then be given by types that have interesting paths, like the circle, which has a non-trivial loop. Coincidentally, the type of all sets is not a set, since its spatial structure is given by the isomorphisms of sets ‚Äî and sets can have non-trivial automorphism groups.

Idea #2: Some types are best understood as representing spaces, with intrinsic homotopic information, which we can investigate using type-theoretic methods. This interpretation is the core idea of ‚Äúsynthetic homotopy theory‚Äù, a whole field of maths!

We conceptualise some types (like the circle mentioned above) as being representatives for spaces, though this does not mean that they have any intrinsically-detectable topology in the sense of ‚Äúopen sets‚Äù-topological spaces. Instead, we think of these types as being an incarnation of spaces √† la homotopy theory, with structure similar to what you‚Äôd find there: Homology groups, cohomology groups, but most easily seen are the homotopy groups. In these types, type-theoretic paths quack like topological paths.

Indeed, any type in homotopy type theory has an associated sequence of homotopy groups, and these behave precisely like the homotopy groups of classical algebraic topology ‚Äî with the difference that the way we compute them in HoTT has a distinctly more ‚Äútypey‚Äù flavour.

This interpretation of types as spaces leads us to look at the statement ‚Äúthere is a type XXX with œÄ1(X)‚âÖZ\pi_1(X) \cong \mathbb{Z}œÄ1‚Äã(X)‚âÖZ ‚Äù and rightly call XXX ‚Äúthe circle‚Äù, even though a topologist might object that our circle is not any subset of R2\mathbb{R}^2R2 . Idea #3: HoTT provides a setting for doing category theory where all constructions are intrinsically isomorphism-invariant. Under this light, types provide the underlying groupoids of categories.

When doing category theory in HoTT, we have some types that naturally arise as the type of objects in a category, such as the type of groups relative to a universe.
